# 达尔文进化框架 — 通用版

一个让任何项目自主进化、只保留改进的系统。

受Karpathy autoresearch启发，应用于：
- AI Agent提示词优化
- 产品文案/设计迭代
- 代码质量提升
- 工作流优化

## 五条铁律

1. **单一可编辑资产** — 一次只改一个文件/模块
2. **双重评估** — 结构分(60%) + 实测分(40%)
3. **棘轮机制** — 改差了立即revert，只留改进
4. **独立评分** — 修改者和评分者必须分离
5. **人在回路** — 每轮优化后暂停，等人类确认

## 使用方式

```bash
# 对任何项目运行进化循环
darwin <project_path> --config darwin.yaml
```
